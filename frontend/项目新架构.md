# 车流监控系统前端项目需求分析与架构说明书

---

## 一、项目概述

本项目为《WEB 开发前端技术基础》大作业，实现一个**离线可用的车流监控系统**。前端需对接本地模拟服务端，通过 Vue 3 + Vite + Pinia + TypeScript 技术栈进行开发。

系统主要功能：

1. 地图显示出入口、路径与实时车辆位置。
2. 实时更新车辆数据并在地图上展示。
3. 车辆历史信息查询及轨迹回放。
4. 按车牌号及出入口统计通行与费用信息。
5. 路径拥挤程度可视化。
6. 离线可用，不依赖网络资源。

---

## 二、功能需求分析

### 1. 地图模块

- **功能**：在 800×600 区域地图上显示出入口及路径。
- **输入**：出入口列表 `/getEntries`。
- **输出**：渲染出入口、路径。
- **交互**：
    - 点击出入口查看详情。
    - 支持地图缩放和平移（加分）。
- **延伸功能**：
    - 热力图叠加显示拥挤程度。
    - 夜间/白天主题切换。

---

### 2. 车辆实时显示模块

- **功能**：显示区域内行驶车辆的实时位置。
- **输入**：
    - `/getVehiclePositions` 获取位置。
    - `/getVehicleDetail` 获取车辆详细信息。
- **输出**：车辆图标，点击显示车牌号、速度等信息。
- **交互**：
    - 鼠标悬停显示详情。
    - 支持实时轨迹显示（加分）。

---

### 3. 历史查询模块

- **功能**：根据车牌号查询历史信息，支持分页。
- **输入**：
    - `/getVehiclesHistory?No={车牌号}&Page={页码}`
- **输出**：
    - 历史记录列表，包括车牌号、日期、费用等。
- **交互**：
    - 点击历史记录回放车辆路径。
- **加分延伸**：
    - 时间段筛选。
    - 导出 CSV/JSON 文件。

---

### 4. 统计分析模块

- **功能**：
    - 按车牌号统计费用： `/getStatisticsByVehicle`
    - 按出入口统计通行总数： `/getStatisticsByEntry`
- **交互**：
    - 时间段筛选。
    - 图表显示统计结果（柱状图、折线图）。
- **延伸功能**：
    - 历史对比分析。
    - 趋势预测（加分）。

---

### 5. 路径拥挤程度模块

- **功能**：根据车辆数量标注路径拥挤等级。
- **输入**：实时车辆位置 + 出入口/检查点信息。
- **输出**：
    - 不同颜色显示等级：
        - 0-3 → 一级
        - 4-6 → 二级
        - 7-9 → 三级
- **延伸功能**：
    - 动态热力图。
    - 拥挤预警提示。

---

### 6. 离线资源模块

- **功能**：保证系统在离线环境可用。
- **措施**：
    - 所有 JS、CSS、图片资源本地存储。
    - Service Worker 可选，缓存 API 数据。
- **加分延伸**：
    - 离线模拟数据演示。
    - Mock 数据模拟全部功能。

---

## 三、技术栈
前端框架：Vue 3 + Vite
状态管理：Pinia（统一管理所有车辆、历史、统计、拥挤度状态）
类型系统：TypeScript
渲染：Canvas 双层（静态背景 + 动态车辆）
性能优化：
requestAnimationFrame 驱动动画
插值算法平滑车辆移动
Canvas 离屏缓存静态层
搜索框防抖、窗口调整节流
WebP 图标资源、离线预加载

## 六、加分延伸功能

## 七、性能与离线优化
- 双层 Canvas：静态层缓存路径与出入口，动态层刷新车辆。
- 插值平滑：线性插值使车辆移动更流畅。
- 资源离线化：JS、CSS、图片全部本地存储。
- 防抖/节流：输入框防抖、窗口大小调整节流。
- Service Worker：可选，缓存 API 返回的 Mock 数据。
