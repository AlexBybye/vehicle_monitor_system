## 1️⃣ 项目分层思路

**建议结构**（src 目录下）：

src/  
│-- assets/             # 静态资源：图标、地图底图、样式  
│-- api/                # 对应后端模拟接口封装  
│   ├-- entries.ts  
│   ├-- vehicles.ts  
│   ├-- statistics.ts  
│-- components/         # 可复用组件  
│   ├-- MapCanvas.vue  
│   ├-- VehicleIcon.vue  
│   ├-- HeatmapOverlay.vue  
│   ├-- SearchBar.vue  
│   └-- ChartPanel.vue  
│-- views/              # 页面级模块  
│   ├-- MapView.vue  
│   ├-- HistoryView.vue  
│   ├-- StatisticsView.vue  
│   └-- CongestionView.vue  
│-- store/              # Pinia 状态管理  
│   └-- trafficStore.ts  
│-- utils/              # 工具函数、插值算法、时间处理等  
│-- types/              # TypeScript 类型定义  
│-- App.vue  
│-- main.ts

**分层说明**：

- **api/**：集中处理所有 fetch/axios 调用，保持业务逻辑与接口分离。
- **store/**：Pinia 统一管理车辆、历史、统计、拥挤度状态，实现全局数据响应。
- **components/**：小粒度可复用组件，如地图、车辆图标、热力图、搜索框等。
- **views/**：每个功能页独立文件，组合 components 与 store，保证页面逻辑清晰。
- **utils/**：动画插值、节流防抖、数据导出 CSV/JSON 等纯函数工具。
- **types/**：统一 TS 类型定义，便于维护接口数据结构。

## 2️⃣ 功能模块对应分文件

|功能模块|文件拆分建议|主要职责|关联 Store/组件|
|---|---|---|---|
|地图显示|`MapCanvas.vue` + `HeatmapOverlay.vue`|渲染出入口、路径，支持缩放、平移、热力图|trafficStore.entries, trafficStore.checkpoints, trafficStore.pathCongestion|
|车辆实时显示|`VehicleIcon.vue` + `MapCanvas.vue`|动态渲染车辆位置、轨迹、悬停详情|trafficStore.vehicles, trafficStore.vehicleDetails|
|历史查询|`HistoryView.vue` + `SearchBar.vue` + `VehicleHistoryTable.vue`|查询分页历史记录，支持轨迹回放、导出|trafficStore.historyRecords|
|统计分析|`StatisticsView.vue` + `ChartPanel.vue`|按车牌/出入口统计，图表显示趋势|trafficStore.statistics|
|路径拥挤程度|`CongestionView.vue` + `HeatmapOverlay.vue`|可视化路径拥挤等级，触发预警|trafficStore.pathCongestion, trafficStore.congestionAlerts|
|离线资源|Vite 配置 + `serviceWorker.ts` + Mock 数据文件|本地 JS/CSS/图片缓存，离线演示|所有模块 API 调用可 fallback 本地 Mock|

## 3️⃣ 前端开发思路（vibe coding）

1. **先做 Store**：
    - 用 Pinia 定义全局状态，保证每个功能模块的数据来源统一。
    - Actions 中封装接口调用 + 数据处理（计算拥挤度、触发预警）。
    - 方便后续各页面直接 `useTrafficStore()` 获取数据。
2. **组件化拆分**：
    - 小组件独立、纯渲染、props 驱动。
    - MapCanvas 负责静态层 + 动态层绘制，内部用 `requestAnimationFrame` 平滑车辆移动。
    - HeatmapOverlay 可复用到地图和拥挤等级页面。
3. **页面组合**：
    - MapView、HistoryView、StatisticsView、CongestionView 各自组合组件 + store。
    - 页面内部只处理视图逻辑，如分页、筛选、导出，避免重复接口调用。
4. **工具函数**：
    - 插值算法、节流防抖、CSV/JSON 导出。
    - 可在 utils 里写 TypeScript 函数，按需引入。
5. **离线模式**：
    - 所有接口先调用 API，失败则 fallback Mock 数据。
    - 静态资源打包到 assets，本地缓存。
    - 可选 Service Worker，实现完全离线浏览。

## 4️⃣ 开发顺序建议（快速可视化反馈）

1. 构建 Pinia store + Mock 数据。
2. 地图 + 车辆实时显示（最核心，能看到动态效果）。
3. 历史查询页（分页 + 回放）。
4. 统计分析页（图表显示）。
5. 路径拥挤可视化（热力图 + 预警）。
6. 离线功能测试。
